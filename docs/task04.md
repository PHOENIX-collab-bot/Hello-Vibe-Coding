## 一、核心目标：给型接入 AI 能力

- 本节核心：**把 AI 能力真正“接入产品”**
- 不只是调用 API，而是形成一个**完整闭环**
- 目标：
  - 选择 AI 能力（LLM / 文生图 / 图生图）
  - 实现前后端交互
  - 集成到已有原型中



## 二、AI 能力接入的基本流程

整体流程可以理解为三步：

### 1. 选择 AI 能力类型

常见能力：

- 文本生成（LLM）
  - 文案生成、对话、总结
- 图像生成（Text-to-Image）
  - 根据描述生成图片
- 图像编辑（Image-to-Image）
  - 改图、风格迁移

本质：根据你的产品需求，选择合适能力



### 2. 调用 AI API

关键点：

- 调用第三方 AI 服务（如模型 API）
- 传入：
  - prompt（提示词）
  - 参数（温度、长度等）
- 获取返回结果（文本 / 图片）

本质：
把“用户输入”转成“模型可理解的输入”



### 3. 前后端打通

典型结构：

- 前端：
  - 用户输入 prompt
  - 展示 AI 输出结果
- 后端：
  - 接收前端请求
  - 调用 AI API
  - 返回结果

流程：

```
用户 → 前端 → 后端 → AI API → 后端 → 前端 → 用户
```



### 4. 集成进产品原型

不是单独 demo，而是：

- 嵌入业务流程中
- 成为功能的一部分

例如：

- 电商：
  - 商品文案生成
  - 商品图生成
- 内容平台：
  - 自动写标题
  - 自动生成封面



## 三、关键理解：不是“功能”，而是“能力组件”

AI 在这里不是一个孤立功能，而是：

 可复用的能力模块

可以被：

- 多个业务节点调用
- 串成完整流程



## 四、从“单点能力”到“完整系统”

下一步目标：

把多个 AI 能力串起来，形成完整业务链路：

示例（电商 AI 工作台）：

1. 内容策划（LLM）
2. 商品图生成（文生图）
3. 图片优化（图像编辑）
4. 数据分析（LLM）



## 五、实践任务（重点）

需要完成一个闭环项目：

- 接入一个 AI 能力
- 完成前后端交互
- 集成到原型中



## 六、关键能力总结（你要掌握的）

这节课本质训练的是：

### 1. API 调用能力

- 会用 AI 接口
- 理解输入输出

### 2. Prompt 设计能力

- 如何让 AI 输出符合预期

### 3. 系统整合能力

- 前端 + 后端 + AI

### 4. 产品思维

- AI 放在哪个业务节点最有价值



## 七、一句话总结

 本节的核心不是“用 AI”，而是
**把 AI 变成产品里的一个可用能力模块，并跑通完整闭环**
